Библиотека для работы с датчиком TCS3200

В библиотеке для подсчета частоты с выходной ноги датчика используется внешнее прерывание INT0 (ножка 2 ардуино). 

Перед началом использования необходимо провести инициализацию датчика методом begin().

Метод loop() в цикле опрашивает состояние компонент RGBC датчика и записывается их актуальное значение во внутренние переменные класса.

Метод getRGB() возвращает текущие значения RGB. В качестве параметра методу необходимо передать объявленный внешне по отношению к классу массив uint8_t размерностью [3].
В результате вызова метода в первом элементе массива [0] будет содержаться компонента "R", во втором [1] - компонента "G", в третьем [2] - компонента "B".

Метод getRGBtoMaxCorrection() аналогичен предыдущему методу, но возвращает текущее значение RGB, усиленное до максимальной яркости. 
Например, если текущее считанное с датчика значение RGB = {123, 180, 65}, то метод getRGBtoMaxCorrection() вернет значение RGB = {174, 255, 92}.
Данная возможность может быть полезна для более легкого распознавания оттенков и для уменьшения влияния расстояния до распознаваемого предмета на результат распознавания.


Следует принять во внимание, что в силу простоты внутренней реализации библиотеки, индивидуальных особенностей отдельного датчика, 
результат измерений будет сильно зависить от расстояния, на котором проводится измерение. Поэтому для четкого различия оттенков следует подбирать цвета, 
в которых одна или несколько комонент различаются значительно (желательная разница не менее 50 единиц).

Также следует понимать, что черный цвет - это полное отсутствие какого-либо излучения (видимого света, инфракрасного, ультрафиолетового, рентгеновского и др. типов излучений).
Т.е. полностью нулевое значение скорее всего соответствует неисправному датчику.

Также следует осторожно пользоваться методом getRGBtoMaxCorrection(). Например, белый цвет и любой оттенок серого вернут одно и то же значение (255, 255, 255).
